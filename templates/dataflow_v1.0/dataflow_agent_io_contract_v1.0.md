# Dataflow Project 数据流图智能体输入输出契约 v1.0（中文翻译版）

## 1. 目的

本契约定义 Dataflow Project 数据流图智能体读取什么输入、执行哪些处理、生成哪些输出。本文为中文翻译版，仅翻译原方案中的文字、字段和文件名，不改变方案内容。

## 2. 输入：数据采集包 DCP

智能体只读取 DevOps 提交的数据采集包：

```text
数据采集包DCP_v1.0/
  dataflow_main_collection_template_v1.0.xlsx
  原始导出/
  证据/
  说明/
```

其中，`dataflow_main_collection_template_v1.0.xlsx` 是唯一结构化输入源。

## 3. 必需工作表

智能体必须识别以下工作表：

```text
00_元数据
01_GCP项目
02_网络
03_服务器
04_服务
05_依赖关系
06_数据资产
07_防火墙
08_CloudArmor
09_IAM服务账号
10_监控
11_交付链路
12_外部服务
13_问题例外
14_证据索引
90_枚举
```

## 4. 通用控制字段

多数采集工作表需要包含以下控制字段：

```text
记录ID
环境
来源类型
证据ID
确认状态
采集人
采集时间
备注
```

确认状态枚举值：

```text
已确认
自动发现
待确认
已拒绝
不适用
```

## 5. 智能体处理流程

```text
读取输入
  ↓
校验字段和工作表
  ↓
校验主键和外键
  ↓
归一化数据
  ↓
生成节点和关系
  ↓
生成 Graph JSON / YAML
  ↓
执行一致性与风险检查
  ↓
生成图
  ↓
生成报告
  ↓
打包最终交付物
```

## 6. 节点生成规则

| 工作表 | 生成节点 |
|---|---|
| 01_GCP项目 | GCP项目节点 |
| 02_网络 | VPC、子网、NAT、LB、PSC、VPC Peering 节点 |
| 03_服务器 | 服务器 / VM 节点 |
| 04_服务 | 服务节点 |
| 06_数据资产 | 数据资产节点 |
| 07_防火墙 | 防火墙规则节点 |
| 08_CloudArmor | Cloud Armor 策略节点 |
| 09_IAM服务账号 | 服务账号 / IAM 绑定节点 |
| 10_监控 | 监控、日志、告警、XDR 覆盖节点 |
| 11_交付链路 | CI/CD 与交付链路节点 |
| 12_外部服务 | 外部服务节点 |

## 7. 关系生成规则

| 关系类型 | 来源 |
|---|---|
| 包含 | GCP项目、网络、服务器之间的归属关系 |
| 运行在 | 服务到服务器的运行关系 |
| 调用 | 服务到服务的通信关系 |
| 读 / 写 | 服务到数据资产的访问关系 |
| 允许 | 防火墙规则与服务/依赖的关联 |
| 防护 | Cloud Armor 与入口对象的关联 |
| 使用服务账号 | 服务与服务账号的关系 |
| 监控覆盖 | 服务/服务器/数据资产与监控对象的关系 |
| 部署到 | CI/CD 流水线与服务或实例的关系 |

## 7.1 可选增强字段

`04_服务` 可以填写运行上下文字段，用于生成 runtime 节点、服务下钻图和边 metadata：

```text
运行类型(Runtime_Type)
运行ID(Runtime_ID)
运行名称(Runtime_Name)
运行命名空间(Runtime_Namespace)
运行集群(Runtime_Cluster)
运行区域(Runtime_Region)
```

`05_依赖关系` 可以填写显式依赖目标字段。当这些字段存在时，智能体优先使用它们解析依赖目标；旧版目标服务、目标外部服务、目标数据资产字段仍然兼容：

```text
目标类型(Target_Type)
目标ID(Target_ID)
交互模式(Interaction_Mode)
```

## 9. 智能体不应执行的动作

智能体不得自动修改生产环境配置，不得自动修改防火墙，不得自动修改 IAM，不得自动执行部署，不得自动接受安全风险。

## 10. 错误处理规则

如果发现字段缺失、外键断裂、关键链路缺失、证据缺失或安全监控异常，智能体应生成问题/风险记录，而不是自动补全不存在的事实。
